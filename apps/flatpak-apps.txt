# flatpak-apps.txt — Recetas de apps instaladas via Flatpak (Flathub)
# ==============================================================================
# Cada linea define una app de Flathub a instalar. El bootstrap (categoria
# 'apps') y la funcion de shell 'flatpak-app <id>' las usan.
#
# Flatpak baja un binario ya armado y crea el .desktop solo; no hay cadena de
# build. Las apps corren aisladas (sandbox) y traen su propio runtime, por eso
# sirven para casos que necesitan Chromium/Electron (p.ej. Teams: lo requiere
# para las videollamadas).
#
# Formato:  id|Nombre|app-id-de-flathub
#   - id      : identificador corto (= el del catalogo del bootstrap)
#   - Nombre  : nombre visible (solo informativo, para el menu de seleccion)
#   - app-id  : ID de la app en Flathub (ej. com.github.IsmaelMartinez.teams_for_linux)
#
# Para agregar una app: suma una linea aca y corre 'flatpak-app <id>' (o
# re-bootstrap eligiendo la app). Lineas con # se ignoran.
#
# NOTA: el bootstrap agrega el remote 'flathub' COMPLETO a nivel usuario. El
# Flathub que trae Fedora viene 'filtered' (censurado) y no lista todas las apps.
# ==============================================================================
teams|Teams for Linux|com.github.IsmaelMartinez.teams_for_linux
